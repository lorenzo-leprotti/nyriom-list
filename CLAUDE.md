# AEROCOM 2025 Delegate Prioritization Pipeline

## Project Overview

**Goal:** Transform ~500 AEROCOM 2025 delegates into a ranked TOP 50 engagement list for Nyriom Technologies (Berlin-based bio-polymer composites startup, NyrionPlex product line).

**Context:** Nyriom sells advanced composite materials to aerospace companies. The ideal prospect either directly selects materials (engineering/procurement roles) or influences material decisions (R&D, sustainability, C-suite). AEROCOM 2025 is a fictional aerospace composites conference in Toulouse.

**Portfolio piece:** This is a demonstration project. Company names (Airbus, Hexcel, Safran, etc.) are real; individual delegate names are generated. The pipeline architecture and scoring logic are production-representative.

---

## Pipeline

```
522 delegates (input/aerocom_2025_delegates.csv)
  │
  ├── step1_buckets.py: Categorize + filter (instant, $0)
  │     Three-layer filter: whitelist (A-G) + role exclusion (bucket 10) + score >= 40
  │     Result: 249 relevant delegates
  │
  ├── step2_research.py: AI research — 13 variables per delegate
  │     Backend: --backend demo | perplexity | ollama
  │
  ├── step3_score.py: Multi-factor scoring + ranking
  │     Title + Company + Materials Proximity + Research Boosts
  │
  ├── step4_enhance.py: Deep research on TOP 60
  │     Backend: --backend demo | perplexity | ollama
  │
  └── step5_final_rank.py: Final ranking → TOP 50 deliverable
        Output: output/AEROCOM_2025_FINAL_TOP50.xlsx (6 sheets)
```

## Architecture

```

research_backends.py      ← Backend abstraction (Perplexity / Ollama / Demo)
```

## Scoring Algorithm

**Priority Score = Title + Company + Materials Proximity + Research Boosts + Key Contact Bonus**

| Component | Max | Source |
|-----------|-----|--------|
| Title Score | 30 | `TITLE_SCORES` in config (CEO=30, VP=22, Director=12, Engineer=5) |
| Company Score | 40 | Company Bucket (D=40, A=35, B=30, C=25, E=20, G=10, F=5) |
| Materials Adoption Proximity | 25 | Title-based + org bonus + research data |
| Research Boosts | 30+ | Lightweighting, sustainability, R&D budget signals |
| Key Contact Bonus | 30 | Pre-identified targets in config |

## Key Design Decisions

1. **Whitelist > blacklist** for company filtering — unknowns are the problem, not known exclusions
2. **`_has_data()` helper** — Perplexity returns "NOT_FOUND - however..." which must NOT trigger score boosts. Uses `startswith("not_found")`
3. **Single-pass default** — Dual-pass verification is optional (`--dual-pass` flag) for cost efficiency
4. **ENHANCE_TOP_N = 60** — Configurable via config; balances coverage vs. cost
5. **Demo data ~43% NOT_FOUND** — Simulates realistic API gaps for authentic scoring behavior
